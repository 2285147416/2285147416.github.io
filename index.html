#概念：

协调节点：coordinator node, 所有节点往该节点发送心跳信息
主节点：primary node, 主要用于跑一些独立流程
独立流程：isolated processors, 有些流程需要独立跑，不要在集群中运行，例如getsftp流程，该流程需要从FTP服务器上获取文件信息，如果多个流程同时运行则会影响文件的获取一致性。

我们定义nifi安装包根目录地址为$NIFI_HOME。例如当前dn1节点中

Zookeeper服务集群配置
Nifi集群的协调配置管理需要第三方协调配置服务，即zookeeper集群。 可以使用环境中已经搭建好的zookeeper服务，NIFI本身也集成了zookeeper服务，我们采用NIFI自身启动的Zookeeper。三个NIFI节点集群，同时也相应搭建三个节点的Zookeeper服务集群。为此需要修改一下配置：
修改$NIFI_HOME/conf/nifi.properties
nifi.state.management.embedded.zookeeper.start=true //启用nifi中的zookeeper服务
nifi.state.management.embedded.zookeeper.properties=./conf/zookeeper.properties //zookeeperf服务集群配置信息，即集群的配置在zookeeper.properties中定义

#集群状态管理
我们知道nifi中所有数据流的处理会涉及到状态保存，例如从FTP取文件，会保存文件的最新时间，以便于下次取数据的时候取最新时间下的文件信息。默认单点情况下数据流的状态都保存在服务节点本地，但是在启用集群后，需要将这些状态信息保存在第三方公共服务区，这样以便集群中所有服务节点可以获取到并且同步状态信息。而这个第三方服务一般是采用zookeeper进行管理。为此需要实现以下几点配置：
nifi.state.management.configuration.file=./conf/state-management.xml 状态管理配置文件 采用默认即可
nifi.state.management.provide.cluster=zk-provider 状态管理采用zk-provider方式

#修改$NIFI_HOME/conf/state-management.xml文件，找到并且id为zk-provider的标签。

修改Connect String 为 dn1:2181,dn2:2181,dn3:2181
修改Root Node 为 /nifi

#集群节点属性配置
修$NIFI_HOME/conf/nifi.properties中以下配置：

nifi.cluster.is.node=true
nifi.cluster.node.address=dn1
nifi.zookeeper.connect.string=dn1:2181,dn2:2181,dn3:2181
nifi.zookeeper.root.node=/nifi
nifi.remote.input.host=dn3
nifi.remote.input.socket.port=9998
Nifi.web.http.host=dn3

STEP SIX 启动各个节点
在各个服务节点上，进入$NIFI_HOME目录，执行bin/nifi.sh start进行各个节点启动。
访问任意节点(以dn1节点为例): http://dn1:8080/nifi

我的第一篇博客文章

这是我的第一篇博客文章,主要是记录如何使用hexo+github搭建个人的博客网站;

以下是一些操作步骤:

  1. 准备

    注意事项

    输入代码时，核对准确，最好切换成英文输入法；
    将文中的 “用户名” 和 “邮箱” 替换为自己的 GitHub 账户名和绑定的邮箱；
    统一使用 Git Bash 进行操作（支持 Win、Mac）；
    小白请严格按步骤进行，不要跳！

    环境搭建

    Hexo 基于 Node.js，搭建过程中还需要使用 npm（Node.js 已带） 和 git，因此先搭建本地操作环境，安装 Node.js 和 Git。

Node.js：https://nodejs.org/zh-cn
Git：https://git-scm.com/downloads
下载 Node.js 和 Git 程序并安装，一路点 “下一步” 按默认配置完成安装。

安装完成后，Win+R 输入 cmd 并打开，依次输入 node -v、npm -v 和 git –version 并回车，如下图出现程序版本号即可。

  1. 连接 Github
    使用邮箱注册 GitHub 账户，选择免费账户（Free），并完成邮件验证。

右键 -> Git Bash Here，设置用户名和邮箱：

git config –global user.name “GitHub 用户名”
git config –global user.email “GitHub 邮箱”
创建 SSH 密匙：

输入 ssh-keygen -t rsa -C “GitHub 邮箱”，然后一路回车。

添加密匙：

进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

  1. 创建 Github Pages 仓库
    GitHub 主页右上角加号 -> New repository：

Repository name 中输入 用户名.github.io
勾选 “Initialize this repository with a README”
Description 选填
填好后点击 Create repository 创建。

  1. 本地安装 Hexo 博客程序
    新建一个文件夹用来存放 Hexo 的程序文件，如 Hexo-Blog。打开该文件夹，右键 -> Git Bash Here。

4.1 安装 Hexo

使用 npm 一键安装 Hexo 博客程序：

npm install -g hexo-cli
Mac 用户需要管理员权限（sudo），运行这条命令：

sudo npm install -g hexo-cli
安装时间有点久（真的很慢！），界面也没任何反应，耐心等待，安装完成后如下图。

4.2 Hexo 初始化和本地预览

初始化并安装所需组件：

hexo init # 初始化
npm install # 安装组件
完成后依次输入下面命令，启动本地服务器进行预览：

hexo g # 生成页面
hexo s # 启动预览
访问 http://localhost:4000，出现 Hexo 默认页面，本地博客安装成功！

  1. 部署 Hexo 到 GitHub Pages
    本地博客测试成功后，就是上传到 GitHub 进行部署，使其能够在网络上访问。

首先安装 hexo-deployer-git：

npm install hexo-deployer-git –save
然后修改 _config.yml 文件末尾的 Deployment 部分，修改成如下：
deploy:
type: git
repository: git@github.com:用户名/用户名.github.io.git
branch: master
完成后运行 hexo d 将网站上传部署到 GitHub Pages。
完成！这时访问我们的 GitHub 域名 https://用户名.github.io 就可以看到 Hexo 网站了。

  1. 开始使用

    7.1 发布文章

    进入博客所在目录，右键打开 Git Bash Here，创建博文：

hexo new “My New Post”
然后 source 文件夹中会出现一个 My New Post.md 文件，就可以使用 Markdown 编辑器在该文件中撰写文章了。

写完后运行下面代码将文章渲染并部署到 GitHub Pages 上完成发布。以后每次发布文章都是这两条命令。

hexo g # 生成页面
hexo d # 部署发布
也可以不使用命令自己创建 .md 文件，只需在文件开头手动加入如下格式 Front-matter 即可，写完后运行 hexo g 和 hexo d 发布。

title: Hello World # 标题
date: 2019/3/26 hh:mm:ss # 时间
categories: # 分类
